# RULE-002: 截图审查规则

> 状态: 生效中
> 创建日期: 2026-01-25

---

## 目的

建立 Claude Code 与用户之间的截图沟通机制，用于反馈 UI 效果、报告问题等场景。

---

## 截图目录

```
data/prtsc/
```

---

## 触发词

当用户消息包含以下关键词时，Claude 应主动查看截图：

- "你看截图"
- "你看看这个效果"
- "看下截图"
- "截图里"
- "prtsc"

---

## 查看规则

1. **倒序查看**：按文件修改时间倒排，优先查看最新截图
2. **增量查看**：只查看本次对话中尚未查看过的新截图
3. **红框关注**：用户会用红色边框/圆圈标记需要关注的区域
4. **数量限制**：单次最多查看 3 张，除非用户明确要求查看更多

---

## 命名格式

截图使用系统默认命名，常见格式：

```
Screenshot YYYY-MM-DD HHMMSS.png
```

示例：
- `Screenshot 2026-01-25 060244.png`

Claude 通过文件名中的时间戳判断顺序。

---

## Claude 响应模板

查看截图后，Claude 应：

1. 描述看到的主要内容
2. 指出红框标记区域的问题或特征
3. 提出改进建议或确认效果符合预期
